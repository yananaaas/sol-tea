## Що це за проект

Концептуальний портфоліо e-commerce магазин нішових китайських чаїв **Söl Tea**. Мета — показати рівень як дизайнера і як Design Engineer: не просто макети, а повністю живий, задеплоєний сайт із власною дизайн системою, кодом і продуманою UX-архітектурою.

Стиль: мінімалістично-editorial, без пафосу, без кліше про "традиції".  
Аудиторія: молода українська аудиторія, яка цінує якість і момент.

## Технічна база

- **Чистий HTML/CSS/JS** — без фреймворків, без npm
- **Деплой:** GitHub Pages (push → ~1 хв до оновлення)
- **Шрифти:** Google Fonts — Inter 900 (заголовки), DM Sans 300–500 (UI)
- **Іконки:** HugeIcons stroke-rounded (CDN), тільки `hgi-stroke`, ніколи `hgi-solid`
- **Стан:** sessionStorage (`solCart`, `solWish`)
- **Дані товарів:** JS-масив `PRODUCTS` в sol-shop.html / sol-product.html (38 позицій)
- **Зображення:** поки плейсхолдери `<div>Фото</div>`, папка `/images/` готується

### Дизайн система (еталон)

Живий документ: **`sol-design-system.html`** — єдиний стандарт для всіх сторінок.  
Містить: палітру, типографіку, компоненти, spacing, правила.  
Будь-який новий компонент або зміна — спочатку узгоджуємо з тим, що є в DS.

## Питання, що потребують рішення

**Дані / бекенд:**  
Зараз всі дані — в JS. Для реального портфоліо як Design Engineer варто розглянути:
- **Notion API** — як CMS для контенту (товари, тексти). Мінус: потрібен проксі-сервер або Netlify Functions
- **Supabase** — повноцінна база даних + автентифікація
- **Formspree / Netlify Forms** — для форм (підписка, замовлення)
Рекомендація: обговорити в окремому чаті з порівнянням варіантів.

**A/B тестування:**  
Для портфоліо достатньо задокументованих варіантів (не обов'язково живий сплітест).  
Варіант A vs Б — показати в кейсі, пояснити рішення.
